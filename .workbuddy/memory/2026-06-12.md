# 2026-06-12 工作日志

## 项目改造：FastAPI Demo → 学校讨论平台后端

### 修复的 Bug
1. **schemas/user.py**: `UserUpdateModel.is_active` 类型从 `str` 改为 `Optional[bool]`
2. **crud/posts.py**: `is_public` 筛选逻辑从 Python 原生 `or` 改为 SQLAlchemy `or_()`，修复了查询不正确的问题
3. **routers/posts.py**: `add_new_post` 现在从 token 获取 `author_uid`，不再信任前端传入
4. **schemas/posts.py**: `PostsCreateModel.author_uid` 改为 Optional，由后端自动填充

### 新增的安全措施
5. **routers/user.py**: `delete_user` 和 `update_user` 路由添加权限校验，仅允许本人或管理员操作

### 新增的模型和功能
6. **Category 板块系统**: model_category.py, schemas/category.py, crud/category.py, routers/category.py
   - Posts 模型添加 `category_id` 外键、`is_top` 置顶字段
   - 帖子查询支持 `category_id` 筛选
   - 板块管理仅管理员可操作
7. **嵌套评论**: Comments 模型添加 `parent_id` 自引用外键，支持楼中楼回复
   - content 从 `String(255)` 改为 `Text`，取消长度限制
   - 查询返回带 nested replies 的树形结构
8. **Like 点赞系统**: model_like.py + 路由 + CRUD
   - 切换式操作（点赞/取消点赞）
   - 查询点赞数、检查是否已点
9. **Bookmark 收藏系统**: model_bookmark.py + 路由 + CRUD
   - 切换式操作
   - 查看我的收藏
10. **Notification 通知系统**: model_notification.py + 路由 + CRUD
    - 支持 reply / like / system 三种类型
    - 分页查询、未读计数、标记已读/全部已读
    - 管理员可发送系统通知
11. **清理 requirements.txt**: 从 111 行精简到 18 行，移除了 Scrapy、PyAutoGUI、pygame 等无关依赖
